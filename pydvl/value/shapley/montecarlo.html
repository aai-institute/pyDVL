montecarlo

Monte Carlo approximations to Shapley Data values.

Note: You probably want to use the common interface provided by compute_shapley_values() instead of directly using the functions in this module.

Exact computation of Shapley value requires \(\mathcal{O}(2^n)\) retrainings of the model. Recall the definition of the value of sample \(i\):

\[v_i = \frac{1}{n} \sum_{S \subseteq D \backslash \{ i \}} \binom{n - 1}{ | S | }^{-1} [U (S \cup \{ i \}) - U (S)] ,\]

where \(D\) is the set of \(n\) indices in the training set, which we identify with the data itself.

To overcome this problem, it is possible to use various forms of sampling from the power set of the training data to obtain a Monte Carlo approximation to the true value. This is done in combinatorial_montecarlo_shapley() and owen_combinatorial_shapley().

Alternatively, employing the reformulation of the expression above as a sum over permutations, one has the implementation in permutation_montecarlo_shapley(), or using an early stopping strategy to adapt computation time truncated_montecarlo_shapley().

Finally, you can consider grouping your data points using GroupedDataset and computing the values of the groups instead.

truncated_montecarlo_shapley(u: Utility, value_tolerance: Optional[float] = None, max_iterations: Optional[int] = None, *, n_jobs: Optional[int] = None, config: ParallelConfig = ParallelConfig(backend='ray', address=None, num_workers=None), progress: bool = False, coordinator_update_period: int = 10, worker_update_period: int = 5) → Tuple[OrderedDict[str, float], Dict[str, float]][source]

Monte Carlo approximation to the Shapley value of data points.

This implements the permutation-based method described in Ghorbani and Zou1. It is a Monte Carlo estimate of the sum over all possible permutations of the index set, with a double stopping criterion.

Warning

This function does not exactly reproduce the stopping criterion of Ghorbani and Zou1 which uses a hardcoded time delay in the sequence of values. Instead, we use a moving average and the stopping criterion detailed in check_done().

Todo

Instead of naively implementing the expectation, we sequentially add points to a dataset from a permutation. We keep sampling permutations and updating all shapley values until the std/value score in the moving average falls below a given threshold (value_tolerance) or when the number of iterations exceeds a certain number (max_iterations).

Parameters

  • u – Utility object with model, data, and scoring function

  • value_tolerance – Terminate if the standard deviation of the average value for every sample has dropped below this value

  • max_iterations – Terminate if the total number of permutations exceeds this number.

  • n_jobs – number of jobs processing permutations. If None, it will be set to available_cpus().

  • config – Object configuring parallel computation, with cluster address, number of cpus, etc.

  • progress – Whether to display progress bars for each job.

  • coordinator_update_period – in seconds. Check status with the job coordinator every so often.

  • worker_update_period – interval in seconds between different updates to and from the coordinator

Returns

Tuple with the first element being an collections.OrderedDict of approximate Shapley values for the indices, and the second being the estimated standard error of each value.

owen_sampling_shapley(u: Utility, max_iterations: int, max_q: int, *, method: OwenAlgorithm = OwenAlgorithm.Standard, n_jobs: int = 1, config: ParallelConfig = ParallelConfig(backend='ray', address=None, num_workers=None), progress: bool = False) → Tuple[OrderedDict[str, float], Dict[str, float]][source]

Owen sampling of Shapley values.

This function computes a Monte Carlo approximation to

\[v_u(i) = \int_0^1 \mathbb{E}_{S \sim P_q(D_{\backslash \{ i \}})} [u(S \cup {i}) - u(S)]\]

as described in Okhrati and Lipani2, using one of two methods. The first one, selected with the argument mode = OwenAlgorithm.Standard, approximates the integral with:

\[\hat{v}_u(i) = \frac{1}{Q M} \sum_{j=0}^Q \sum_{m=1}^M [u(S^{(q_j)}_m \cup {i}) - u(S^{(q_j)}_m)],\]

where \(q_j = \frac{j}{Q} \in [0,1]\) and the sets \(S^{(q_j)}\) are such that a sample \(x \in S^{(q_j)}\) if a draw from a \(Ber(q_j)\) distribution is 1.

The second method, selected with the argument mode = OwenAlgorithm.Anthithetic, uses correlated samples in the inner sum to reduce the variance:

\[\hat{v}_u(i) = \frac{1}{Q M} \sum_{j=0}^Q \sum_{m=1}^M [u(S^{(q_j)}_m \cup {i}) - u(S^{(q_j)}_m) + u((S^{(q_j)}_m)^c \cup {i}) - u((S^{(q_j)}_m)^c)],\]

where now \(q_j = \frac{j}{2Q} \in [0,\frac{1}{2}]\), and \(S^c\) is the complement of \(S\).

Warning

Antithetic sampling is unstable and not properly tested

Parameters

  • u – Utility object holding data, model and scoring function.

  • max_iterations – Numer of sets to sample for each value of q

  • max_q – Number of subdivisions for q ∈ [0,1] (the element sampling probability) used to approximate the outer integral.

  • method – Selects the algorithm to use, see the description. Either Full for \(q \in [0,1]\) or Halved for \(q \in [0,0.5]\) and correlated samples

  • n_jobs – Number of parallel jobs to use. Each worker receives a chunk of the total of max_q values for q.

  • config – Object configuring parallel computation, with cluster address, number of cpus, etc.

  • progress – Whether to display progress bars for each job.

Returns

Tuple with the first element being an ordered Dict of approximate Shapley values for the indices, the second being their standard error
