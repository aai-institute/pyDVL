montecarloÔÉÅ

Monte Carlo approximations to Shapley Data values.

Note: You probably want to use the common interface provided by compute_shapley_values() instead of using the functions in this module.

Exact computation of Shapley value requires \(\mathcal{O}(2^n)\) retrainings of the model. Recall the definition of the value of sample \(i\):

\[v_i = \frac{1}{N} \sum_{S \subseteq D_{\backslash \{ i \}}} \frac{1}{\binom{N - 1}{ | S | }} [U (S_{\cup \{ i \}}) - U (S)] ,\]

where \(D\) is the set of indices in the training set, which we identify with the data itself.

To overcome this limitation, it is possible to only sample some subsets of the training set (or permutations thereof) to obtain a Monte Carlo approximation to the true value. This is done in combinatorial_montecarlo_shapley(). Alternatively, employing the reformulation of the expression above as a sum over permutations, one has the implementation in permutation_montecarlo_shapley().

Additionally, one can implement an early stopping strategy to adapt computation time. This is done in truncated_montecarlo_shapley().

Finally, you can consider grouping your data points using GroupedDataset and computing the values of the groups instead.

MonteCarlo approximation to the Shapley value of data points.

This implements the method described in:

Ghorbani, Amirata, and James Zou. ‚ÄòData Shapley: Equitable Valuation of Data for Machine Learning‚Äô. In International Conference on Machine Learning, 2242‚Äì51. PMLR, 2019. http://proceedings.mlr.press/v97/ghorbani19c.html.

Instead of naively implementing the expectation, we sequentially add points to a dataset from a permutation. We keep sampling permutations and updating all shapley values until the std/value score in the moving average falls below a given threshold (value_tolerance) or when the number of iterations exceeds a certain number (max_iterations).

Parameters

  • u ‚Äì Utility object with model, data, and scoring function

  • value_tolerance ‚Äì Terminate if the standard deviation of the average value for every sample has dropped below this value

  • max_iterations ‚Äì Terminate if the total number of permutations exceeds this number.

  • n_jobs ‚Äì number of jobs processing permutations. If None, it will be set to available_cpus().

  • config ‚Äì Object configuring parallel computation, with cluster address, number of cpus, etc.

  • progress ‚Äì set to True to use tqdm progress bars.

  • coordinator_update_frequency ‚Äì in seconds. Check status with the job coordinator every so often.

  • worker_update_frequency ‚Äì interval in seconds between different updates to and from the coordinator

Returns

Tuple with the first element being an collections.OrderedDict of approximate Shapley values for the indices, and the second being the estimated standard error of each value.

permutation_montecarlo_shapley(u: Utility, max_iterations: int, n_jobs: int, config: ParallelConfig = ParallelConfig(backend='ray', address=None, num_workers=None), *, progress: bool = False) → Tuple[OrderedDict[str, float], Dict[str, float]][source]ÔÉÅ

Computes an approximate Shapley value by sampling independent index permutations to approximate the sum:

\[v_u(x_i) = \frac{1}{n!} \sum_{\sigma \in \Pi(n)} [u(\sigma_{i-1} \cup {i}) ‚àí u(\sigma_{i})].\]

See Computing data values for details.

Parameters

  • u ‚Äì Utility object with model, data, and scoring function.

  • max_iterations ‚Äì total number of iterations (permutations) to use across all jobs.

  • n_jobs ‚Äì number of jobs across which to distribute the computation.

  • config ‚Äì Object configuring parallel computation, with cluster address, number of cpus, etc.

  • progress ‚Äì Set to True to print a progress bar.

Returns

Tuple with the first element being an ordered Dict of approximate Shapley values for the indices, the second being their standard error.

combinatorial_montecarlo_shapley(u: Utility, max_iterations: int, n_jobs: int = 1, config: ParallelConfig = ParallelConfig(backend='ray', address=None, num_workers=None), *, dist: PowerSetDistribution = PowerSetDistribution.WEIGHTED, progress: bool = False) → Tuple[OrderedDict[str, float], Dict[str, float]][source]ÔÉÅ

Computes an approximate Shapley value using the combinatorial definition:

\[v_u(i) = \frac{1}{n} \sum_{S \subseteq N \setminus \{i\}} \binom{n-1}{ | S | }^{-1} [u(S \cup \{i\}) ‚àí u(S)]\]

See Computing data values for details.

Parallelization is done by splitting the set of indices across processes and computing the sum over subsets \(S \subseteq N \setminus \{i\}\) separately.

Parameters

  • u ‚Äì utility

  • max_iterations ‚Äì total number of subsets to use for each index. Any number much larger than \(2^{n-1}\) is thus wasteful. Note that this has different semantics from the homonymous parameter for permutation_montecarlo_shapley().

  • n_jobs ‚Äì number of jobs across which to distribute the computation

  • config ‚Äì Object configuring parallel computation, with cluster address, number of cpus, etc.

  • dist ‚Äì Distribution to use of sets over the power set.

  • progress ‚Äì true to plot progress bar

Returns

Tuple with the first element being an ordered Dict of approximate Shapley values for the indices, the second being their standard error
